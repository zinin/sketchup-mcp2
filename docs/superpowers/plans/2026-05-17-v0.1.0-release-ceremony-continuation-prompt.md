### Version bump policy (for release notes)

- `MIN_RUBY` / `MAX_RUBY` (Python) and `MIN_PYTHON` / `MAX_PYTHON` (Ruby) all bumped to `0.1.0..0.1.0` (matched pair).
- **Reason:** the one-time `hello` handshake is a breaking wire-level change. Old `sketchup-mcp2 <= 0.0.3` Python clients lack the handshake entirely; they would send a per-request `client_version` field which the new server rejects with `-32600 "first method must be 'hello'"`.

## v0.1.0 — multi-client server + one-time handshake

First **signed** release.

### Breaking changes

- **Wire protocol: one-time `hello` handshake on connect.** Every TCP
  connection must begin with a JSON-RPC `hello` request carrying
  `params.client_version`; server replies with `server_version` +
  `client_id`. Per-request / per-response version envelopes are
  removed. Old Python clients (`sketchup-mcp2 <= 0.0.3`) fail the
  handshake with `-32600` on the first frame; upgrade both the Python
  package and the `.rbz` together.

### New features

- **Multi-client TCP server** — N concurrent MCP clients can connect
  to the SketchUp extension simultaneously; per-client error
  isolation; FIFO frame dispatch on the SketchUp UI thread.
- **`get_viewport_screenshot` MCP tool** — captures the SketchUp
  viewport as PNG (returns an MCP `Image`; requires SketchUp 2026+).
- **`sketchup_modeling_strategy` MCP prompt** — surfaced in MCP-aware
  clients' slash menu.
- **Settings dialog** — `Plugins → MCP Server → Settings...` for host /
  port / log level (persisted in SketchUp preferences under section
  `SU_MCP`).
- **Modular Ruby architecture** — `core/`, `handlers/`, `helpers/`,
  `ui/` (replaces the previous monolithic plugin file).
- **First Trimble-signed `.rbz`** — appears as "Signed" in Extension
  Manager.

### Install

1. SketchUp extension: install `su_mcp_v0.1.0.rbz` via
   `Window → Extension Manager → Install Extension` and restart.
2. Python MCP server: `uvx sketchup-mcp2` (will be published to PyPI
   shortly if not already).

### Compatibility

- SketchUp 2026+ for `get_viewport_screenshot`; SketchUp 2017+ for
  everything else.
- Python 3.10+.
