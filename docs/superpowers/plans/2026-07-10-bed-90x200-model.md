**Goal:** Построить в живой модели SketchUp две кровати под матрас 900×2000×200 (A — 2 ящика + изголовье, B — 3 ящика) по спеке `docs/superpowers/specs/2026-07-10-bed-90x200-design.md`, каждая деталь — именованная группа с проверенным bbox.

## Global Constraints

- Все размеры — **мм**; в Ruby API только через деление на 25.4 (`make_box` конвертирует сам; вручную не конвертировать).
- Каждый чанк **обязан** сам оборачивать правки: `model.start_operation(...)` → `commit_operation`, с `abort_operation` в rescue. `eval_ruby` этого НЕ делает. Операции не вкладывать.
- Чанки самодостаточны: `eval_ruby` исполняется в `TOPLEVEL_BINDING.dup`, локальные переменные и лямбды между вызовами НЕ сохраняются — прелюдию повторять в каждом чанке целиком.
- Имена групп — строго из этого плана (совпадают со спекой §5–6). Материалы: `MCP Pine`, `MCP Plywood`, `MCP Metal`, `MCP Mattress`.
- Существующие сущности пользователя не трогать; `model.save` не вызывать; удалять что-либо — только по явному подтверждению пользователя.
- Верификация: возвращённый JSON сравнивать с таблицей Expected таска; расхождение любой координаты > 0.2 мм = провал шага.
- Recovery при провале: один вызов MCP-инструмента `undo` (откатывает весь чанк — одну операцию), исправить скрипт, повторить чанк. Не «допиливать» модель дополнительными правками.
- Один чанк = один вызов `eval_ruby`. Скрипты подавать без сокращений, включая прелюдию.

## Прелюдия (повторяется в начале КАЖДОГО строительного чанка)

Ниже в тасках она обозначена комментарием `# --- PRELUDE --- (вставить целиком блок «Прелюдия» из docs/superpowers/plans/2026-07-10-bed-90x200-model.md)`.
При исполнении заменять этот комментарий полным текстом:

```ruby
require "json"
mm = 25.4
model = Sketchup.active_model

def make_box(parent_ents, x_mm, y_mm, z_mm, w_mm, d_mm, h_mm)
  mm = 25.4
  grp = parent_ents.add_group
  face = grp.entities.add_face(
    [x_mm/mm,          y_mm/mm,          z_mm/mm],
    [(x_mm + w_mm)/mm, y_mm/mm,          z_mm/mm],
    [(x_mm + w_mm)/mm, (y_mm + d_mm)/mm, z_mm/mm],
    [x_mm/mm,          (y_mm + d_mm)/mm, z_mm/mm]
  )
  sign = face.normal.z >= 0 ? 1 : -1
  face.pushpull(sign * h_mm / mm)
  grp
end

matl = lambda do |name, r, g, b|
  m = model.materials[name] || model.materials.add(name)
  m.color = Sketchup::Color.new(r, g, b)
  m
end
pine  = matl.call("MCP Pine",     230, 199, 154)
ply   = matl.call("MCP Plywood", 240, 221, 176)
metal = matl.call("MCP Metal",   158, 164, 171)
mgray = matl.call("MCP Mattress", 217, 217, 222)

part = lambda do |ents, name, material, x, y, z, w, d, h|
  g = make_box(ents, x, y, z, w, d, h)
  g.name = name
  g.material = material
  g
end

gbox = lambda do |g|
  b = g.bounds
  [b.min.x, b.min.y, b.min.z, b.max.x, b.max.y, b.max.z].map { |v| (v * mm).round(1) }
end

report = lambda do |node|
  h = {}
  node.entities.grep(Sketchup::Group).each { |g| h[g.name] = gbox.call(g) }
  h[node.name] = gbox.call(node)
  h
end
```

Пояснения: `make_box(ents, x, y, z, w, d, h)` — куб от угла (x,y,z) размером w×d×h мм (рецепт из cookbook, защищён от инверсии нормали); `def` персистентен между вызовами, но повторное определение безвредно. `matl` идемпотентен (`materials.add` с существующим именем не вызывается повторно).

## Примечание для исполнителя

- Порядок Tasks 2–10 взаимонезависим геометрически (узлы не пересекаются), но выполнять их по порядку — так проще откатывать хвост при ошибке (`undo` откатывает последнюю операцию).
- Каждый вызов `eval_ruby` проходит per-call review в MCP-клиенте — это ожидаемо; не пытаться объединять все чанки в один вызов.
- Если пользователь работает в модели с существующей геометрией — она не мешает: все наши группы имеют уникальные имена (проверено в Task 1).
